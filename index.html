Selma

Self Managing Software Defined Networks

Download this project as a .zip file Download this project as a tar.gz file

Abstract.

A key reason for the tremendous success of traditional networks is self management (plug-and-play MAC learning, routing around failures with default OSPF configuration, automated load balancing with ECMP, etc.). We argue that to facilitate large-scale adoption of softwaredefined networking (SDN), the same level of self management must be provided. We present SelMa, a selfmanaging platform for SDNs. SelMa automatically and proactively generates network configurations that adhere to operator-specified abstract policies (access control, traffic steering, etc.) and dynamically adapts configuration in response to policy/topology changes. To accomplish this, SelMa leverages new algorithmic insights to detect policy conflicts, compose policies, and optimize the translation of abstract global policies to concrete lowlevel forwarding state configuration. We evaluate a software implementation of SelMa for large numbers of network policies and on a broad variety of networks.

Running the System.

If you wish to run Selma on your own network all you have to do is the following: 1. Download "full solution" directory to a windows computer.

  1. Edit the graph file (test.graph) as follows:

  • The first row is the number of nodes (switches or routers) in your graph. The nodes will be identified by a number between 0 and this number minus one.

  • After a blank row each row represents a directed edge with its weight. For example a directed edge from node "1" to node "2" with weight 4 will be written: "1 2 4".

  1. Edit the policies input file (policies_input.py) as follows:
  2. Give names to ip subnets.
  3. Add each network function to FUNCTIONS_AND_AREAS map, and state where it resides. An integer represents the node in the graph, and a tuple an undirected edge.
  4. Add groups to the GROUPS map, by stating which ip subnets they include.
  5. Put in the IP_LOCATIONS map the location (node) of each subnet.
  6. Add policies as shown in the file to the POLICIES list.
